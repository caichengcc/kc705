Xylon OS Library: xyl_oslib for standalone (possix) Xilinx BSPs

xyl_oslib standalone (possix) specifics:

  • Xylon OS library for standalone (BareMetal) or simple threading systems like Xilinx Xilkernel.
  • Specific to this variant of the xyl_oslib library is the SwConfig mechanism. Swconfig files must be compiled to each application and they implement following functionality: timers (getTime, sleep, ..), malloc/free, lock/unlock (in Posix systems this is related to Mutex ). For template for the SWConfig files see: SwConfig templates
  • there are several compile time options available when compilinf the xyl_oslib from the SDK:
    • some options ara visible from the SDK "Board Support Package Settings" window:
      • OSLIB_CPU_SUPPORT - support for the direct CPU access to memory/registers - enabled by default
      • OSLIB_CPU_VLINK_SUPPORT - support for the Xylon VLINK bus access - disabled by default
      • OSLIB_FILE_SUPPORT - support for particular file-system - disabled by default
    • some options are set directly in the command line (in the CPU extra_compiler_flags):
      • NDEBUG   (-DNDEBUG) - disables all debug code and debug/error prints, used for release builds
      • OSLIB_ADD_DEBUG (-DOSLIB_ADD_DEBUG) - adds debug/error prints, can be used in combination with NDEBUG
Using xyl_oslib from Xilinx SDK:
  • xyl_oslib is integrated with the Xilinx SDK using Xilinx Libgen mechanisms (data folder: mdd, tcl, src folder: makefile)
  • To insert the xyl_oslib in the SDK BSP project:
    •  you must add the search path to SDK: Xilinxtools->Repositories->New (add path to folder that is two levels before xyl_oslib folder, e.g. if you have /user/hardware/sw_services/xyl_oslib_v_1_6_a then add path to/user/hardware ), click on the Refresh Repositories in SDK
    • SDK should now be able to locate the xyl_oslib in the BSP Settings window
